### 英文全称

- Network Exposure Function

### 中文释义

- 网络能力开放功能

### 解释

NEF 是 5GC（5G核心网）的核心网元之一，负责将 5G 网络的能力（如用户位置、QoS 策略、网络切片状态等）安全地开放给第三方应用（如车联网平台、工业互联网平台），实现网络能力的商业化变现。

### NEF 可开放的网络能力

| 能力类型 | 说明 | 应用场景 |
|---------|------|---------|
| **用户位置** | 实时位置查询 | 车联网、智慧物流 |
| **QoS 信息** | 当前业务的 [[QoS]] 状态 | 视频加速服务 |
| **切片状态** | 各切片资源使用情况 | 切片租赁管理 |
| **流量路由** | 用户流量路径信息 | 分流服务 |
| **策略控制** | 网络策略查询和配置 | 垂直行业专网 |
| **计费信息** | 用量计费数据 | 差异化计费 |

### NEF 安全机制

| 机制 | 说明 |
|------|------|
| **AF 认证** | 第三方应用需通过 NEF 认证才能调用能力 |
| **能力过滤** | NEF 根据应用授权，只开放允许的能力 |
| **流量监控** | 监控 API 调用，防止滥用 |
| **数据脱敏** | 敏感数据脱敏处理后再开放 |

### NEF 与 OneM2M / 第三方平台集成

```
第三方应用 → NEF → 5GC 内部 NF (AMF/SMF/PCF)
                ↓
           能力开放 API
                ↓
           垂直行业平台
```

### 相关概念

- [[5GC]] - NEF 所在的 5G 核心网
- [[SBI]] - NEF 对外提供 RESTful API
- [[SMF]] - 提供 QoS/计费相关信息
- [[AMF]] - 提供用户位置和移动性信息
- [[MEC]] - NEF 与 MEC 协同实现低时延能力开放

> 反向链接: [[MEC]] 通过 NEF 获取网络信息 ([[QoS]]、位置)，实现基于网络能力的创新应用，如 V2X [[V2X]] 中的意图导航
