### 英文全称

- File Transfer Protocol

### 中文释义

- 文件传输协议

### 解释

FTP 是 TCP/IP 协议族中的应用层协议，用于在网络中实现文件的上传和下载。在通信行业中，FTP 广泛用于网管系统采集性能数据、上传设备日志、备份配置文件等场景。

### FTP 工作原理

| 组件 | 说明 |
|------|------|
| **FTP Server** | 文件服务器，提供文件存储服务 |
| **FTP Client** | 客户端，发起文件传输请求 |
| **控制连接 (Port 21)** | 传输命令和响应 |
| **数据连接 (Port 20/动态)** | 传输实际文件数据 |

### FTP 传输模式

| 模式 | 说明 | 数据传输端口 |
|------|------|-------------|
| **主动模式 (PORT)** | 客户端开放端口等待服务器连接 | 客户端指定端口 |
| **被动模式 (PASV)** | 服务器开放端口等待客户端连接 | 服务器指定端口 |

### FTP 在通信行业的应用

| 场景 | 说明 |
|------|------|
| **网管数据采集** | [[NMS]]/[[EMS]] 通过 FTP 采集 [[KPI]] 数据 |
| **日志上传** | 基站设备定时上传日志到日志服务器 |
| **配置备份** | [[gNB]]/[[eNB]] 配置定期备份到 FTP 服务器 |
| **版本文件下发** | 软件升级包通过 FTP 下发到网元 |
| **MR/MDT 数据** | 用户测量报告导出到 FTP |

### FTP 配置示例（华为网管）

| 参数 | 说明 | 示例 |
|------|------|------|
| **服务器地址** | FTP 服务器 IP | 192.168.1.100 |
| **用户名** | 认证用户名 |ftpuser |
| **密码** | 认证密码 | ******** |
| **上传路径** | 文件上传目录 | /data/kpi |
| **采集周期** | 数据采集间隔 | 每 5 分钟 |

### 相关概念

- [[NMS]] / [[EMS]] - 网管系统，常用 FTP 采集数据
- [[KPI]] - 性能数据通过 FTP 采集
- [[TSDB]] - FTP 采集的数据最终存入 TSDB
- [[OMC]] - 操作维护中心，可能使用 FTP

> 反向链接: [[NMS]] 通过 FTP 从 [[gNB]]/[[eNB]] 采集 [[KPI]] 和日志数据，是网管数据采集的主要协议之一
