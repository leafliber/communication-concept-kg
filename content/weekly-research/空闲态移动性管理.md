# 空闲态移动性管理

## 概述

空闲态移动性管理（Idle Mode Mobility Management）是 5G 系统中 UE（用户设备）在 **RRC_IDLE** 状态下，自主完成小区选择和小区重选的一系列机制。与连接态由网络侧控制切换不同，空闲态的移动性决策主要由 **UE 自主执行**，网络侧通过系统消息广播参数来引导 UE 的行为。

空闲态移动性管理的核心目标：
1. **驻留最优小区** — 保证 UE 始终驻留在信号质量最好的合适小区
2. **节省终端功耗** — 尽量减少不必要的测量和信令交互
3. **保证网络可达性** — 确保 UE 在被呼叫时能被寻呼到
4. **负载均衡** — 通过重选参数引导 UE 分布到不同频点/小区

---

## RRC 状态与空闲态定位

### 5G RRC 三态模型

5G NR 相比 4G LTE 新增了 **RRC_INACTIVE** 状态，形成三态模型：

```
                    RRC Connection Release
         ┌──────────────────────────────────────┐
         │                                      │
         ▼                                      │
   ┌───────────┐   RRC Resume    ┌───────────┐  │
   │ RRC_IDLE  │───────────────►│RRC_INACTIVE│  │
   │  空闲态    │                │  去激活态    │  │
   └─────┬─────┘◄───────────────└──────┬──────┘  │
         │     RRC Suspend/Release      │         │
         │                              │         │
         │    RRC Setup/Resume          │         │
         ▼                              ▼         │
   ┌─────────────────────────────────────────┐    │
   │            RRC_CONNECTED                 │    │
   │              连接态                       │────┘
   └─────────────────────────────────────────┘
```

| 状态 | UE 行为 | 网络侧 | 移动性方式 |
|------|---------|--------|-----------|
| **RRC_IDLE** | 监听寻呼、测量邻区 | 释放所有 UE 上下文 | **小区选择/重选（UE 自主）** |
| **RRC_INACTIVE** | 监听寻呼、测量邻区 | 保留 UE 上下文（RNA 管理） | 小区重选 + RNA 更新 |
| **RRC_CONNECTED** | 数据收发、测量上报 | 维持完整 UE 上下文 | **切换（网络控制）** |

### 空闲态与连接态移动性的核心区别

| 维度 | 空闲态 | 连接态 |
|------|--------|--------|
| **决策方** | UE 自主 | 网络侧（gNodeB） |
| **机制** | 小区选择 + 小区重选 | 测量上报 + 切换执行 |
| **信令** | 无需信令交互（除位置更新） | 需要测量配置、切换命令等 |
| **目标** | 选择驻留小区 | 保证业务连续性 |
| **速度** | 较慢（秒级） | 较快（毫秒级） |

## 空闲态移动性的两大核心机制

### 一、小区选择（Cell Selection）

小区选择发生在 UE 开机、从覆盖盲区恢复、或从连接态释放进入空闲态时。

#### 1.1 小区选择流程

```
UE 开机 / 恢复覆盖
        │
        ▼
┌─────────────────────┐
│  步骤1: PLMN 选择    │
│  选择可用 PLMN       │
└─────────┬───────────┘
          │
          ▼
┌─────────────────────┐
│  步骤2: 频率扫描      │
│  搜索选择 PLMN 对应   │
│  的可用频点           │
└─────────┬───────────┘
          │
          ▼
┌─────────────────────┐
│  步骤3: 小区搜索      │
│  PSS/SSS 同步        │
│  解码 PBCH → SSB     │
│  解码 SIB1           │
└─────────┬───────────┘
          │
          ▼
┌─────────────────────┐
│  步骤4: S 准则判决    │
│  Srxlev > 0 且       │
│  Squal > 0 ?        │
└────┬──────────┬──────┘
     │ YES      │ NO
     ▼          ▼
  驻留成功    尝试下一个小区
```

#### 1.2 S 准则（S Criterion）

UE 通过 S 准则判断一个小区是否"合适"（Suitable Cell）：

**Srxlev（接收电平满足度）：**
```
Srxlev = Qrxlevmeas - Qrxlevmin - Qrxlevminoffset
         - Pcompensation - Qoffsettemp    (dB)
```

**Squal（接收质量满足度）：**
```
Squal = Qualmeas - Qualmin - Qualminoffset
        - Qoffsettemp                       (dB)
```

| 参数 | 含义 |
|------|------|
| `Qrxlevmeas` | UE 测量到的小区 RSRP |
| `Qrxlevmin` | 最低接收电平要求（SIB1 中广播） |
| `Qrxlevminoffset` | 偏置量（用于漫游场景防止乒乓） |
| `Pcompensation` | 功率补偿 = max(PEMAX - PUMAX, 0) |
| `Qualmeas` | UE 测量到的小区 RSRQ |
| `Qualmin` | 最低接收质量要求 |
| `Qoffsettemp` | 临时偏置（连接释放后一段时间内适用） |

**小区合适条件**：`Srxlev > 0` **且** `Squal > 0`

#### 1.3 小区选择模式

| 模式 | 描述 |
|------|------|
| **初始选择** | UE 无先验信息，扫描所有支持的频点 |
| **存储信息选择** | UE 利用之前存储的载波信息和小区信息加速搜索 |

### 二、小区重选（Cell Reselection）

当 UE 已驻留一个小区后，会持续评估邻区质量，如果发现更好的小区就触发重选。这是空闲态移动性的核心。

#### 2.1 小区重选流程

```
UE 正常驻留（Camp Normally）
        │
        ▼
┌─────────────────────────────────────────┐
│          测量规则（基于 Srxlev）           │
│                                         │
│  ┌─ Srxlev > SIntraSearchP            │──► 只测同频            │
│  ├─ SIntraSearchQ < Squal 且           │──► 同频 + 异频         │
│  │   Srxlev > SNonIntraSearchP         │                       │
│  └─ 其他                                │──► 同频 + 异频 + 异系统 │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│           小区排序（R 准则）              │
│                                         │
│  R_s = Qmeas,s + Qhyst                 │  (服务小区)            │
│  R_n = Qmeas,n - Qoffset              │  (邻区)                │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│         重选判决条件                      │
│                                         │
│  同频/等优先级异频：                      │
│    R_n > R_s 持续 Treselection 时间      │
│                                         │
│  高优先级异频：                           │
│    Srxlev > ThreshX,High 持续            │
│    Treselection 时间                     │
│                                         │
│  低优先级异频：                           │
│    服务小区 Srxlev < ThreshServing,Low   │
│    且目标 Srxlev > ThreshX,Low           │
│    持续 Treselection 时间                │
└────────────────┬────────────────────────┘
```

#### 2.2 R 准则（R Criterion）

```
Rs = Qmeas,s + Qhyst           （服务小区排名值）
Rn = Qmeas,n - Qoffset         （邻小区排名值）
```

| 参数 | 含义 |
|------|------|
| `Qmeas,s` | 服务小区的 RSRP 测量值 |
| `Qhyst` | 迟滞值，防止乒乓重选（SIB2/SIB3 广播） |
| `Qmeas,n` | 邻小区的 RSRP 测量值 |
| `Qoffset` | 邻区偏置 = QoffsetFreq + QoffsetCell |

**重选条件**：`Rn > Rs` 持续 `Treselection` 时间（通常 1-4 秒）

#### 2.3 基于优先级的小区重选

5G NR 中异频/异系统重选采用**基于优先级**的机制，网络通过 SIB2/SIB3/SIB4/SIB5 为不同频点分配优先级。

```
优先级高 → 优先驻留
┌────────────────────────────────────────────────┐
│  高优先级频点                                    │
│  条件：目标小区 Srxlev > ThreshX,High            │
│  → 只要满足就重选（不管服务小区多好）              │
├────────────────────────────────────────────────┤
│  同优先级频点（含同频）                           │
│  条件：R_n > R_s 持续 Treselection               │
│  → 谁好用谁                                     │
├────────────────────────────────────────────────┤
│  低优先级频点                                    │
│  条件：服务小区 Srxlev < ThreshServing,Low       │
│        且目标 Srxlev > ThreshX,Low               │
│  → 服务小区差到一定程度才重选                     │
└────────────────────────────────────────────────┘
```

> ⚠️ **注意**：UE 只能重选到比当前驻留频点**更高**或**同等**优先级的频点。只有当服务小区信号质量低于门限时，才允许重选到**更低**优先级的频点。

#### 2.4 同频重选 vs 异频重选

| 维度 | 同频重选 | 异频重选 |
|------|---------|---------|
| **触发** | 基于 R 准则 | 基于优先级 + R 准则 |
| **参数来源** | SIB2 + SIB3 | SIB2 + SIB4 |
| **测量影响** | 无额外功耗（已在工作频率） | 需要异频 GAP 测量，增加功耗 |
| **Qoffset** | QoffsetCell | QoffsetFreq + QoffsetCell |
| **典型场景** | 同频段连续覆盖 | 异频段分层组网（如 2.6G + 700M） |

## 空闲态移动性关键参数速查

### 系统消息与参数映射

| 系统消息 | 携带的关键参数 | 用途 |
|---------|---------------|------|
| **SIB1** | Qrxlevmin, Qqualmin | 小区选择 S 准则基准 |
| **SIB2** | Qhyst, Treselection, 优先级, 测量门限 | 重选公共参数 |
| **SIB3** | 同频邻区信息, QoffsetCell | 同频重选 |
| **SIB4** | 异频邻区信息, QoffsetFreq, ThreshX | 异频重选 |
| **SIB5** | 异系统邻区信息 | 跨系统重选（NR↔LTE） |

### 核心参数详解

| 参数 | 典型值 | 调优方向 |
|------|--------|---------|
| **Qhyst**（迟滞） | 2-6 dB | ↑ 防乒乓但降低重选灵敏度 |
| **Treselection**（重选时间） | 1-4 s | ↑ 防乒乓但增加重选时延 |
| **QoffsetFreq**（频点偏置） | 0-24 dB | 引导 UE 优先驻留目标频点 |
| **QoffsetCell**（小区偏置） | 0-24 dB | 微调特定邻区的重选倾向 |
| **ThreshX,High/Low** | 20-60 dBm | 异频重选门限 |
| **SIntraSearchP** | 30-60 dBm | 同频测量启动门限 |

## 空闲态移动性管理在网优中的实际应用

### 场景一：异频负载均衡

**目标**：让空闲态 UE 更均匀地分布在多个频点上，避免某频点负载过高。

**方法**：
- 给负载轻的频点设置**更高优先级**
- 调低 ThreshX,High 使 UE 更容易重选到高优先级频点
- 调高 QoffsetFreq 降低当前频点的吸引力

### 场景二：700M 深覆盖策略

**目标**：700M 频段覆盖好但容量小，应作为底层覆盖网，只在信号差时才让 UE 驻留。

**方法**：
- 700M 设为**低优先级**
- 调低 ThreshServing,Low，让 UE 尽量留在中高频
- 只有当中高频信号真的很差时才重选到 700M

### 场景三：高铁场景优化

**目标**：高速移动时 UE 频繁重选导致位置更新信令风暴。

**方法**：
- 增大 Qhyst（如 8-12 dB）
- 增大 Treselection（如 3-5 s）
- 减少不必要的邻区配置
- 使用高速场景专用参数集

### 场景四：室内外协同

**目标**：让室内用户优先驻留室分小区，减少宏站干扰。

**方法**：
- 室分频点设置**更高优先级**
- 调低 ThreshX,High 使 UE 进入室内后快速重选到室分
- 室分边缘适当增大 QoffsetCell 防止乒乓

## 空闲态 → 连接态的衔接

空闲态重选决策会直接影响后续连接态体验：

```
空闲态驻留小区 ──RRC 建立──► 连接态初始小区
                                │
                    如果质量差 → 立即触发切换
```

**关键影响**：
- UE 在空闲态驻留到信号差的小区 → RRC 建立成功率降低
- 频繁重选 → 位置更新信令量大 → 影响核心网性能
- 重选参数设置不当 → 连接态切换压力增大（UE 初始接入就在不合适的小区）

因此，**空闲态移动性参数优化是连接态移动性优化（切换优化）的基础**。空闲态做好了，连接态的很多问题可以从源头减少。

> 📖 参考文档中（SA组网连接态移动性管理，第 3.1 节 RRC 状态）详细描述了 RRC 三态模型及其转换条件，空闲态与连接态的移动性管理是一个完整体系的两面。

## 与连接态移动性管理的对比总结

| 维度 | 空闲态移动性 | 连接态移动性 |
|------|------------|------------|
| **触发** | UE 自主（基于广播参数） | 网络控制（基于测量报告） |
| **核心机制** | 小区选择（S准则）+ 小区重选（R准则/优先级） | 测量事件（A1-A6）+ 切换执行 |
| **测量** | UE 按 SIB 配置自主测量 | gNB 下发测量控制，UE 上报 |
| **判决** | UE 本地计算 | gNB 计算 |
| **执行** | 直接驻留新小区 + 位置更新 | HO Command → 随机接入 → 路径切换 |
| **中断时间** | 无（本身无业务） | 有（ms 级切换中断） |
| **网优关注点** | 重选参数、邻区配置、优先级设置 | 切换参数、邻区关系、切换策略 |
| **参数来源** | SIB 广播 | RRC 重配置（专用信令） |
