### 什么是干扰分析

干扰分析是移动通信网络优化中的核心工作，通过分析网络中的干扰信号来源、类型和影响，制定干扰排查和治理方案，提升网络质量 ([[KPI]]) 和用户体验。

### 干扰类型分类

#### 1. 按干扰来源

| 类型 | 来源 | 说明 |
|------|------|------|
| **系统内干扰** | 同系统其他小区 | 同频干扰，最常见 |
| **系统间干扰** | 其他系统 | 异系统干扰 (如 4G/5G 间) |
| **外部干扰** | 非通信设备 | 干扰器、屏蔽器、私装放大器 |

#### 2. 按干扰方向

| 类型 | 说明 | 典型场景 |
|------|------|---------|
| **下行干扰** | 基站→终端方向 | 屏蔽器、下行同频干扰 |
| **上行干扰** | 终端→基站方向 | 私装放大器上行干扰 |
| **上下行干扰** | 双向 | TDD 系统的上下行干扰 |

#### 3. 按频率关系

| 类型 | 频率关系 | 处理难度 |
|------|---------|---------|
| **同频干扰** | 相同频段 | 最严重，需优化 |
| **邻频干扰** | 相邻频段 | 通过保护带缓解 |
| **互调干扰** | 多频非线性组合 | 难以排查 |

### 5G TDD 上下行干扰

TDD 系统最大的挑战是上下行干扰：

```
┌─────────────────────────────────────────────────────────────┐
│               5G TDD 系统间干扰场景                           │
│                                                             │
│  小区 A (上行)              小区 B (下行)                    │
│       ↑                         ↓                           │
│    手机发射                   基站发射                       │
│       │      ←── 交叉时隙干扰 ──→ │                          │
│  强干扰信号  ──────────────→ 接收灵敏度恶化                    │
└─────────────────────────────────────────────────────────────┘
```

| 干扰场景 | 说明 | 解决方案 |
|---------|------|---------|
| **小区间干扰** | 相邻小区 TDD 配置不同步 | [[时钟同步]]、干扰协调 |
| **覆盖重叠** | 相邻小区覆盖重叠区 | 功率控制、切换参数优化 |
| **特殊子帧配置** | 保护间隔不够 | 调整 GP、上行预占用 |

### 干扰分析指标

| 指标 | 说明 | 正常范围 |
|------|------|---------|
| **[[SINR]]** | 信号与干扰加噪声比 | >5 dB (一般), >15 dB (良好) |
| **[[RSRP]]** | 参考信号接收功率 | -85 ~ -105 dBm (视场景) |
| **NRT (Noise Rise)**
| 上行干扰底噪抬升 | <6 dB |
| **PRB 利用率** | 干扰时 PRB 利用率高但吞吐低 | - |

### 干扰排查方法

| 方法 | 说明 |
|------|------|
| **后台 KPI 分析** | 分析干扰相关 [[KPI]] (掉话、切换失败) |
| **[[MR]] 数据分析** | 用户 [[UE]] 上报的测量报告 |
| **路测 (DT/CQT)** | 现场测试定位干扰区域 |
| **频谱扫描** | 使用频谱仪扫描干扰信号 |
| **告警分析** | 查看基站是否有干扰相关告警 |

### 常见干扰场景及解决方案

| 场景 | 干扰类型 | 解决方案 |
|------|---------|---------|
| **NSA 锚点干扰** | 4G/5G 站址共址 | 优化锚点优先级、切换参数 |
| **TDD 上下行干扰** | 相邻小区时隙不同步 | 统一 TDD 上下行配置 [[时钟同步]] |
| **私装放大器** | 上行干扰 | 排查覆盖黑洞、安装屏蔽器 |
| **设备故障** | 器件故障导致干扰 | 更换 [[AAU]]/[[RRU]]/[[天馈系统]] |
| **模三干扰** | PCI 冲突导致 | 优化 [[PCI]] 配置 |

### 干扰与 KPI 关系

| 干扰程度 | [[SINR]] | 用户感知 |
|---------|---------|---------|
| **无干扰** | >15 dB | 满速下载，流畅视频 |
| **轻度干扰** | 5-15 dB | 速率下降，可能卡顿 |
| **中度干扰** | 0-5 dB | 频繁重传，视频缓冲 |
| **重度干扰** | <0 dB | 掉线、脱网 |

### 相关概念

- [[SINR]] - 干扰分析的核心指标
- [[KPI]] - 干扰影响网络 KPI
- [[MR]] / [[MRS]] - 干扰分析的测量数据来源
- [[PCI]] - PCI 冲突导致同频干扰
- [[时钟同步]] - TDD 上下行干扰的根本原因
- [[覆盖优化]] - 通过覆盖优化减少干扰

> 反向链接: [[SINR]] 是判断干扰程度的核心指标，当 [[SINR]] 低且 [[RSRP]] 正常时，通常表明存在干扰 ([[网络维护]] 中的重点排查对象)
