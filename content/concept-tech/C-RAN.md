### 英文全称

- Centralized Radio Access Network

### 中文释义

- 云化无线接入网络 / 集中式无线接入网

### 解释

C-RAN 是将传统分布式基站（[[BBU]] + [[RRU]]/[[AAU]]）中的 BBU 集中部署在云端机房，形成 BBU 资源池，多个基站的基带处理资源共享，实现降低能耗、简化运维、提升资源效率的目标。

### C-RAN vs 传统 D-RAN

| 特性 | C-RAN (集中式) | D-RAN (传统分布式) |
|------|--------------|-------------------|
| **BBU 位置** | 集中部署在云端机房 | 部署在每个基站站点 |
| **资源池化** | BBU 资源共享，按需分配 | 各 BBU 独立，使用率低 |
| **光纤需求** | 前传距离可达 10-20 km | 前传距离受限于 300m-1 km |
| **运维成本** | 集中运维，成本降低 | 分散运维，成本高 |
| **能耗** | 资源池共享，夜间可休眠 | 每站独立，夜间无法休眠 |
| **时延** | 增加前传时延 | 无额外时延 |
| **弹性扩缩** | 按需分配资源 | 固定容量 |

### C-RAN 关键技术

| 技术 | 说明 |
|------|------|
| **前传接口 (Fronthaul)** | BBU 与 RRU 间的接口，常用 CPRI/eCPRI |
| **BBU 池化** | 多个小区共享 BBU 资源，提高利用率 |
| **虚拟化** | BBU 功能虚拟化为软件，运行在通用服务器上 |
| **动态调度** | 根据业务负载，动态分配 BBU 资源 |

### 前传接口对比

| 接口 | 速率 | 时延 | 说明 |
|------|------|------|------|
| **CPRI** | 10+ Gbps/载波 | ~0.1 ms | 最早的标准化前传接口 |
| **eCPRI** | 可变 (按需) | ~0.1 ms | 更高效，支持部分功能分离 |
| **IEEE 802.1CM** | 可变 | ~0.1 ms | O-RAN 开放前传标准 |

### C-RAN 的挑战

| 挑战 | 说明 |
|------|------|
| **光纤资源** | 需要大量光纤连接 BBU 池与站点 |
| **前传时延** | 光纤距离增加时延，对 URLLC 业务有影响 |
| **同步** | BBU 集中后，需高精度时钟同步 ([[时钟同步]]) |
| **可靠性** | BBU 池故障影响多个基站，需容灾备份 |

### C-RAN 与 [[O-RAN]] 的关系

| 维度 | C-RAN | [[O-RAN]] |
|------|-------|----------|
| **关注点** | BBU 集中化 | 接口开放、多供应商 |
| **结合** | C-RAN 可基于 O-RAN 架构实现 | O-RAN 的 CU/DU 分离天然支持 C-RAN |

### 相关概念

- [[BBU]] - C-RAN 将 BBU 从站点集中到云端
- [[AAU]] / [[RRU]] - C-RAN 的远端射频单元
- [[gNB]] - C-RAN 可用于 无线接入
- [[O-RAN]] - 开放接口，支持 C-RAN 部署
- [[承载网]] - C-RAN 对承载网光纤资源要求高

> 反向链接: C-RAN 通过集中 BBU 形成资源池，[[BBU]] 可同时服务多个 RRU/AAU 站点，提升资源利用率并降低运维成本
