### 英文全称

- Authentication Server Function

### 中文释义

- 认证服务器功能

### 解释

AUSF 是 5GC（5G核心网）的核心网元之一，负责对 5G 用户（UE）进行身份认证，是 5G 安全架构的第一道防线。支持 5G AKA（认证与密钥协议）等安全机制。

### AUSF 在 5GC 架构中的位置

```
┌─────────────────────────────────────────────────────┐
│                      5GC                              │
│                                                      │
│    ┌─────────┐    ┌─────────┐    ┌─────────┐        │
│    │  [[UDM]]  │←──→│  AUSF  │←──→│  [[AMF]]  │        │
│    │(用户数据)│    │(认证服务)│    │(接入管理)│        │
│    └─────────┘    └────┬────┘    └─────────┘        │
│                        │                              │
│                  ┌─────┴─────┐                        │
│                  │   UE (终端) │                        │
│                  └────────────┘                        │
└─────────────────────────────────────────────────────┘
```

### 5G 认证流程 (5G AKA)

| 步骤 | 说明 |
|------|------|
| 1 | [[AMF]] 向 AUSF 发起认证请求 |
| 2 | AUSF 向 [[UDM]] 请求认证数据 |
| 3 | UDM 返回认证向量 (AV) |
| 4 | AUSF 将 AV 转发给 [[AMF]] |
| 5 | [[AMF]] 与 UE 完成双向认证 |
| 6 | 认证成功后，[[AMF]] 与 UE 建立安全上下文 |

### 5G 安全机制特点

| 特性 | 说明 |
|------|------|
| **SUCI 加密** | 用户永久标识符 SUPI 在空口加密传输，防止追踪 |
| **双向认证** | 不仅网络认证终端，终端也认证网络 |
| **完整性保护** | 所有信令消息均完整性保护 |
| **密钥层次化** | 5G 安全密钥层次比 4G 更深 (K→5G-AKA→KAU→NAS 安全上下文) |

### 与 4G LTE 认证对比

| 特性 | 5G AKA (AUSF) | LTE AKA (MME) |
|------|--------------|---------------|
| **认证协议** | 5G AKA | EPS AKA |
| **用户标识** | SUCI (加密) | IMSI (明文，可被追踪) |
| **网络安全** | 支持切片级安全隔离 | 无切片安全 |
| **终端认证网络** | 支持 (5G 新增) | 不支持 |
| **归属地认证** | 可由 AUSF 触发归属地 [[UDM]] 认证 | 需切换到归属 MME |

### 相关概念

- [[5GC]] - AUSF 所在的 5G 核心网
- [[AMF]] - 与 AUSF 协同完成用户认证
- [[UDM]] - 提供 AUSF 所需的认证数据 (AV)
- [[SBI]] - AUSF 对外提供服务化接口
- [[NRF]] - AUSF 的服务注册中心

> 反向链接: [[AMF]] 通过调用 AUSF 的认证服务，完成与 UE 的双向认证后，才允许 UE 接入网络 ([[NSA]]/[[SA]] 均需认证)
