### 英文全称

- Unified Data Management

### 中文释义

- 统一数据管理

### 解释

UDM 是 5GC（5G核心网）的核心网元之一，负责存储和管理 5G 用户的签约数据（Subscription Data），为 [[AMF]] 和 [[SMF]] 提供用户数据查询服务。

### UDM 在 5GC 架构中的位置

```
                    ┌─────────┐
                    │  UDM   │ ← 用户签约数据存储
                    └────┬────┘
                         │
          ┌──────────────┼──────────────┐
          │              │              │
    ┌─────┴─────┐  ┌─────┴─────┐  ┌─────┴─────┐
    │   AMF    │  │   SMF    │  │   AUSF   │  ← 查询/认证
    │(接入管理) │  │(会话管理) │  │(认证服务) │
    └───────────┘  └───────────┘  └───────────┘
```

### 核心功能

| 功能 | 说明 |
|------|------|
| **用户签约数据存储** | 存储用户身份、服务授权、切片订阅等 |
| **签约数据下发** | 向 [[AMF]] 提供用户上下文选择 ([[AMF]] Pool 场景) |
| **签约数据下发** | 向 [[SMF]] 提供会话管理所需的签约信息 |
| **隐式注册/去注册** | 管理用户在 [[AMF]] 的注册状态 |
| **NF 发现** | 与 [[NRF]] 交互，获取可用的 [[SMF]] 实例 |

### 签约数据结构

| 数据项 | 说明 |
|--------|------|
| **SUCI** | 隐藏用户永久标识符 (5G 安全机制) |
| **GPSI** | 通用公共用户标识符 (类似 MSISDN) |
| **Allowed NSSAI** | 用户允许接入的网络切片列表 |
| **Service Area Restriction** | 服务区域限制 |
| **PDU Session 签约** | 用户可建立的 PDU Session 类型和数量 |
| **Subscribed [[QoS]]** | 用户签约的 QoS 策略 ([[QCI]]/5QI) |

### 与 4G PCRF/UDR 的对比

| 特性 | UDM (5GC) | PCRF + UDR (EPC) |
|------|-----------|------------------|
| **架构** | 解耦的 NF，可独立扩缩容 | PCRF 集中处理，UDR 数据存储 |
| **服务化** | 基于 [[SBI]] (HTTP/2 API) | Diameter/Restful 接口 |
| **签约数据** | 结构化存储，支持切片级签约 | 较粗粒度的策略 |

### 相关概念

- [[5GC]] - UDM 所在的 5G 核心网
- [[AMF]] - 查询 UDM 获取用户上下文
- [[SMF]] - 查询 UDM 获取会话签约信息
- [[AUSF]] - 与 UDM 协同完成用户认证
- [[NRF]] - UDM 的服务注册中心
- [[SBI]] - UDM 对外提供服务化接口

> 反向链接: [[AMF]] 从 UDM 获取用户签约数据后，决定用户可接入的切片 ([[NSSF]]) 和 [[QoS]] 策略
