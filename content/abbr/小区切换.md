## 概述

小区切换是指用户在移动过程中，从一个小区（Cell）移动到另一个小区时，无线连接从源小区转移到目标小区的过程。是保证用户业务连续性的基本机制。

## 切换类型

### 根据场景分类

| 类型 | 说明 |
|------|------|
| 同频切换 | 源小区与目标小区同一频段 |
| 异频切换 | 不同频段间切换（需测量） |
| 系统间切换 | 4G↔5G、4G↔2G/3G 等 |

### 根据触发方式

| 类型 | 说明 |
|------|------|
| 基于覆盖 | 覆盖变差触发 |
| 基于负载 | 负载均衡触发 |
| 基于业务 | 业务需求触发 |
| 基于战略 | 手动配置触发 |

## 切换流程

```
1. 测量控制：基站下发测量配置
2. 测量报告：UE 上报邻区测量结果
3. 切换判决：基站根据 A3/A4 等事件判决
4. 切换执行：UE 接入目标小区
5. 切换完成：释放源小区资源
```

## 关键测量事件

| 事件 | 说明 |
|------|------|
| A1 | 服务小区高于门限 |
| A2 | 服务小区低于门限 |
| A3 | 邻区好于服务小区（+偏置） |
| A4 | 邻区高于门限 |
| A5 | 服务小区低于门限 1 且邻区高于门限 2 |
