### 什么是容量规划

容量规划是根据业务需求和用户增长预测，合理配置无线网络资源（基站、载波、频谱、天线等），确保网络能够满足未来一段时间内的业务增长需求，同时避免资源浪费。

### 容量规划核心要素

| 要素 | 说明 |
|------|------|
| **用户数预测** | 未来用户数、终端类型 |
| **业务模型** | VoNR/Video/Data 各业务占比 |
| **频谱资源** | 可用频段、带宽 |
| **[[KPI]] 要求** | 速率、时延、可靠性要求 |
| **覆盖约束** | 覆盖与容量的平衡 |

### 容量评估关键指标

| 指标 | 说明 | 规划目标 |
|------|------|---------|
| **[[PRB]] 利用率** | 无线资源使用率 | <70% (繁忙时段) |
| **平均用户速率** | 每用户可用吞吐量 | 满足业务需求 |
| **[[PDSCH]] 吞吐量** | 下行容量 | 根据 [[MCS]] 估算 |
| **连接用户数** | 单小区同时接入用户数 | 视基站能力 |
| **[[CCE]] 利用率** | 控制信道资源 | <80% |

### 容量规划流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    容量规划流程                                    │
│                                                                 │
│  1. 需求分析                                                     │
│     └─ 业务类型、用户数预测、KPI 要求                              │
│                                                                 │
│  2. 覆盖规划                                                     │
│     └─ 确定基站布局、站间距、覆盖范围                              │
│                                                                 │
│  3. 容量估算                                                     │
│     └─ 基于链路预算、频谱效率估算单站容量                          │
│                                                                 │
│  4. 方案设计                                                     │
│     └─ 载波配置、CA、[[Massive MIMO]] 参数                        │
│                                                                 │
│  5. 仿真验证                                                     │
│     └─ 蒙特卡洛仿真验证容量                                        │
│                                                                 │
│  6. 调整优化                                                     │
│     └─ 根据仿真结果调整参数                                        │
└─────────────────────────────────────────────────────────────────┘
```

### 5G 容量影响因素

| 因素 | 影响 | 说明 |
|------|------|------|
| **频段** | FR2 (mmWave) 容量 > FR1 | 高频带宽大但覆盖小 |
| **带宽** | 带宽越大容量越高 | 20MHz vs 100MHz |
| **[[MIMO]] 层数** | 层数越多容量越高 | SU-MIMO / MU-MIMO |
| **[[MCS]]** | 高阶调制提升峰值 | 256QAM vs 64QAM |
| **[[PRB]] 数** | PRB 数决定资源总量 | 资源调度基础 |
| **用户分布** | 边缘用户降低平均吞吐量 | 覆盖与容量需平衡 |

### 载波聚合 (CA) 容量规划

| 聚合方式 | 说明 | 容量提升 |
|---------|------|---------|
| **带内连续 CA** | 同一频段连续载波聚合 | 2-3 倍 |
| **带间非连续 CA** | 不同频段载波聚合 | 取决于频段组合 |
| **[[3CC]]** | 三载波聚合 | 3 倍 |

### 扩容策略

| 策略 | 适用场景 | 成本 |
|------|---------|------|
| **加站** | 容量缺口大 | 高 |
| **加小区** | 同站址增加小区 | 中 |
| **加载波** | 同小区增加载波 | 中低 |
| **CA 扩频** | 聚合更多载波 | 低 |
| **[[Massive MIMO]] 升级** | 提升频谱效率 | 中 |
| **[[小区切换]]参数优化** | 优化负载均衡 | 低 |

### 容量与覆盖的平衡

| 策略 | 容量 | 覆盖 |
|------|------|------|
| **提升发射功率** | 略提升 | 扩大 |
| **降低天线挂高** | 提升 | 收缩 |
| **增加 [[AAU]] 通道数** | 大幅提升 | 不变 |
| **缩小小区半径** | 提升 | 缩小 |

### 相关概念

- [[KPI]] - 容量规划的目标
- [[PRB]] - 容量规划的核心资源单位
- [[MCS]] - 调制阶数影响容量
- [[Massive MIMO]] - 提升频谱效率
- [[载波聚合]] / [[CA]] - 扩容手段
- [[覆盖优化]] - 容量与覆盖需协同

> 反向链接: 容量规划需要综合考虑 [[PRB]] 利用率、用户分布、频谱效率等因素，通过 [[CA]]、增加载波、[[小区切换]]参数优化等手段平衡覆盖与容量
