### 什么是时钟同步

时钟同步是指通信网络中各个网元（基站、核心网设备、终端）保持一致的时间基准，是 5G TDD（时分双工）系统、载波聚合、多点协作等关键技术正常运行的基础。

### 为什么 5G 需要高精度时钟同步

| 技术场景 | 时钟精度要求 | 后果（同步不良） |
|---------|------------|----------------|
| **TDD 上下行切换** | ±1.5 μs（部分地区） | 上下行干扰，容量下降 |
| **载波聚合 (CA)** | ±500 ns | 聚合失败，速率下降 |
| **多点协作 (CoMP)** | ±500 ns | 用户体验波动 |
| **[[Massive MIMO]]** | ±1.5 μs | 波束指向错误，覆盖收缩 |
| **[[URLLC]]** | <1 μs | 时延超出标准，无法保障 |
| **[[OTN]]/[[IPRAN]] 回传** | <100 ns | 网络层面 KPI 劣化 |

### 同步精度指标

| 标准 | 同步等级 | 精度要求 | 适用场景 |
|------|---------|---------|---------|
| **G.8273.2** | Type A | ±100 ns | 5G 承载网 |
| **G.8273.2** | Type B | ±50 ns | 前传网络 |
| **G.8273.4** | Free-run | ±15 μs | 不需要同步 |
| **G.8273.4** | Holdover | ±4 μs (24h) | 短暂失去参考后维持 |
| **3GPP** | - | ±1.5 μs | TDD 系统 |

### 同步协议

| 协议 | 说明 | 应用 |
|------|------|------|
| **PTP (IEEE 1588v2)** | 精密时间协议，端到端精确同步 | [[IPRAN]]、承载网 |
| **SyncE** | 同步以太网，通过物理层恢复时钟 | 承载网链路 |
| **GPS/北斗** | 卫星定位时间源 | 基站本地时钟参考 |
| **NTP** | 网络时间协议，精度较低 | 网管系统 |
| **G.8273.4** | 电信级时间同步框架 | [[OTN]] |

### 同步部署方式

| 方式 | 说明 | 精度 | 成本 |
|------|------|------|------|
| **GPS/北斗 1PPS + ToD** | 基站配置 GNSS 接收器 | 50-100 ns | 中等 |
| **1588v2 (BC)** | 承载网传递同步 | ±100 ns | 较高 |
| **SyncE** | 以太网物理层时钟恢复 | ±50 ns | 中等 |
| **1588v2 + SyncE 混合** | 综合同步方案 | 更高 | 最高 |
| **Holdover** | 失去外部参考后，用本地时钟维持 | ±4 μs/24h | - |

### 5G TDD 同步的重要性

5G TDD 使用相同频段分离上下行：

```
┌────────────────────────────────────────┐
│         5G TDD 帧结构 (以 n78 为例)      │
├────────────────────────────────────────┤
│  子帧 0 │ 子帧 1 │ 子帧 2 │ 子帧 3 │... │
│ (下行)  │(保护)  │(上行)  │(下行)  │     │
├────────────────────────────────────────┤
│         ↑                               │
│    上下行切换点 (GP: Guard Period)       │
│    需要严格的时间同步保障                │
└────────────────────────────────────────┘
```

- **同步不良** → 上下行信号干扰 → 掉线、吞吐量下降
- **多小区干扰** → 相邻小区使用相同 TDD 配置时，上下行干扰

### 同步故障影响

| 故障现象 | 可能原因 |
|---------|---------|
| 5G TDD 小区上行干扰 | 相邻小区时钟偏差 >1.5 μs |
| 切换成功率低 | 目标小区时钟偏差大 |
| [[VoNR]] MOS 分低 | 语音包时延抖动大 |
| [[Massive MIMO]] 波束偏移 | 多天线相位不同步 |
