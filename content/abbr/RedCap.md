### 英文全称

- Reduced Capability

### 中文释义

- 轻量化 5G 终端 / 降低能力的终端设备

### 解释

RedCap 是 3GPP Rel-17 引入的 5G NR 终端类型，针对中等复杂度、低成本的物联网设备，在保持 5G 特性的同时大幅降低终端复杂度和成本。

### 为什么需要 RedCap

| 设备类型 | 原 5G 终端问题 | RedCap 解决方案 |
|---------|--------------|----------------|
| **可穿戴设备** (智能手表、手环) | 成本高、功耗大 | 降低带宽/天线数，缩减成本 |
| **工业传感器** | 不需要极致性能 | 简化协议栈，支持 5G 特性 |
| **视频监控** (低端摄像头) | 成本过高 | 单接收天线，降低复杂度 |
| **电力物联网** | 4G 能力不足但 5G 太贵 | RedCap 提供折中方案 |

### RedCap vs eMTC/NB-IoT vs 传统 5G

| 特性 | RedCap (Rel-17) | eMTC/NB-IoT (4G) | 传统 5G (eMBB) |
|------|----------------|-----------------|---------------|
| **峰值速率** | ~150 Mbps | ~1 Mbps | 10+ Gbps |
| **带宽** | 20 MHz (FR1) | 1.4-5 MHz | 100+ MHz |
| **天线数** | 1T1R / 1T2R | 1T1R | 64T64R |
| **调制** | 64QAM (可选 256QAM) | 64QAM | 256QAM |
| **功耗** | 中等 | 超低功耗 | 高 |
| **成本** | ~$20-30 | ~$5-10 | >$100 |
| **复杂度** | 中等 | 低 | 高 |
| **支持 [[切片]]** | 是 | 否 | 是 |
| **支持 [[URLLC]]** | 部分 | 否 | 是 |

### RedCap 终端能力等级

| 等级 | 接收天线数 | 最大带宽 | 典型场景 |
|------|-----------|---------|---------|
| **RC.1** | 1T1R | 20 MHz | 可穿戴设备 |
| **RC.2** | 1T2R | 20 MHz | 视频监控、工业传感器 |

### RedCap 关键特性

| 特性 | 说明 |
|------|------|
| **减少接收天线** | 从 2 接收简化为 1 接收 |
| **降低带宽** | 最大 20 MHz（传统 5G 可达 100-400 MHz） |
| **简化双工** | 可采用半双工 FDD，降低终端成本 |
| **降低调制要求** | 可选 64QAM（传统 5G 要求 256QAM） |
| **省电机制** | 支持 [[DRX]]、增强的省电模式 |
| **覆盖增强** | 与传统 5G 共用 [[小区切换]]/重选机制 |

### RedCap 频段支持

- 主要在 FR1 (Sub-6 GHz) 频段
- 常见部署频段：n1, n3, n7, n28, n41, n78, n79
- 未来可能扩展到毫米波 (FR2)

### 应用场景

| 场景 | 说明 | 需求匹配 |
|------|------|---------|
| **智能手表/手环** | 健康监测、消息通知 | 中等速率、低功耗 |
| **视频监控** | 1080P 监控回传 | 10-50 Mbps，适合 RedCap |
| **工业传感器** | 温度、压力、振动传感 | 低速率、无需极致时延 |
| **电力配网** | 配电自动化、用电信息采集 | 高可靠、可维护性 |
| **车联网 (后装)** | OBD 盒子、行车记录仪 | 中等复杂度 |

### 相关概念

- [[NR]] - RedCap 基于 NR 协议栈
- [[5GC]] - 核心网支持 RedCap 终端接入
- [[gNB]] - 无线接入网支持 RedCap 终端
- [[mMTC]] - 与 eMTC/NB-IoT 在物联网场景竞争
- [[DRX]] - 省电机制
- [[QoS]] - RedCap 切片可配置不同 QoS

> 反向链接: [[mMTC]] 是 5G 的海量连接场景，RedCap 在中等复杂度物联网设备上是 eMTC/NB-IoT 的 5G 升级替代方案
