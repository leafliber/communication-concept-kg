### 什么是网络切片

网络切片（Network Slicing）是 5G 时代最重要的技术创新之一。它允许在同一物理网络基础设施上，创建多个端到端的、逻辑上相互隔离的虚拟网络。每个切片拥有独立的资源、配置和 QoS 策略，为不同类型的业务提供定制化的网络服务。

### 为什么需要网络切片

| 业务场景 | 需求特点 | 传统方案的问题 |
|---------|---------|--------------|
| **[[URLLC]]** | 超低时延(<1ms)、超高可靠 | 4G 网络时延，无法保障 |
| **[[eMBB]]** | 大带宽(>100Mbps)、高速率 | 4G 容量不足 |
| **[[mMTC]]** | 海量连接、低功耗 | 4G 无法支撑亿级连接 |
| **行业专网** | 安全保障、定制化 | 需独立建网，成本极高 |

5G 通过网络切片，在一张物理网络上同时满足以上所有需求，大幅降低建网成本。

### 网络切片架构

```
┌─────────────────────────────────────────────────────────┐
│                    物理基础设施                           │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐    │
│  │   无线接入网   │  │   承载网     │  │   核心网     │    │
│  │  (gNB/AAU)  │  │  (IPRAN/PTN) │  │   (5GC)    │    │
│  └─────────────┘  └─────────────┘  └─────────────┘    │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                   端到端切片 (E2E Slice)                  │
├─────────────────┬─────────────────┬─────────────────────┤
│  eMBB 切片       │   URLLC 切片     │    mMTC 切片         │
│  (大带宽业务)    │  (超低时延)      │   (海量连接)         │
│  峰值: 10Gbps   │  时延: <1ms     │  连接: 100万/km²    │
│  QCI: 8/9       │  QCI: 65/68     │  QCI: 8/9           │
└─────────────────┴─────────────────┴─────────────────────┘
```

### 切片的核心要素

| 要素 | 说明 |
|------|------|
| **S-NSSAI** | 切片唯一标识符 (Single Network Slice Selection Assistance Information) |
| **NSSF** | 网络切片选择功能，负责为 UE 选择合适的切片 |
| **AMF 切片** | [[AMF]] 可同时服务多个 Slice |
| **SMF/UPF 切片** | 按需独立部署，支持差异化 [[QoS]] |

### S-NSSAI 示例

| S-NSSAI | 切片类型 | 说明 |
|---------|---------|------|
| 000001 | eMBB | 增强移动宽带切片 |
| 000002 | URLLC | 超可靠低时延切片 |
| 000003 | mMTC | 海量机器通信切片 |
| 01XXXXXX | 专网切片 | 运营商自定义专网切片 |

### 5G 三大场景与切片对应

| 场景 | 切片 | 关键技术 | 典型应用 |
|------|------|---------|---------|
| **eMBB** | eMBB 切片 | 大带宽、载波聚合、256QAM | 4K/8K 视频、云游戏、VR/AR |
| **URLLC** | URLLC 切片 | 短时隙、URLLC HARQ、核心网下沉 | 自动驾驶、工业控制、远程医疗 |
| **mMTC** | mMTC 切片 | PSM/DRX、eDRX、省电机制 | 智慧城市、智能抄表、资产追踪 |

### 切片的生命周期管理

| 阶段 | 说明 |
|------|------|
| **创建** | 运营商通过管理面创建切片实例 |
| **配置** | 分配无线/承载/核心网资源，设置 [[QoS]] 策略 |
| **监控** | 实时监控切片 KPI（如时延、吞吐量、连接数） |
| **调整** | 根据业务需求动态扩缩容 |
| **销毁** | 业务终止后释放切片资源 |

### 切片与行业专网

| 类型 | 说明 | 部署方式 |
|------|------|---------|
| **公网切片** | 运营商向公众用户提供 | 共享基础设施 |
| **专网切片** | 企业租用专用切片资源 | 部分共享或完全隔离 |
| **混合专网** | 核心网部分功能下沉到企业侧 | 灵活定制 |
| **独立专网** | 完全独立的物理网络 | 成本最高（如 5G 专网） |

### 相关概念
