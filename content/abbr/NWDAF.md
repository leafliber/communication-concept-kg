### 英文全称

- Network Data Analytics Function

### 中文释义

- 网络数据分析功能

### 解释

NWDAF 是 5GC（5G核心网）的核心网元之一，负责收集网络运行数据，进行智能分析和预测，为网络切片资源分配、负载均衡、故障检测等提供 AI/ML 驱动的决策支持。

### 核心功能

| 功能 | 说明 |
|------|------|
| **网络数据分析** | 收集实时网络状态、KPI、用户行为等数据 |
| **切片负载分析** | 分析各切片的资源使用情况，预测拥塞 |
| **用户体验分析** | 分析用户 [[QoS]] 体验，识别质差用户 |
| **异常检测** | 基于 AI 检测网络异常和故障 |
| **预测性维护** | 预测设备故障，提前维护 |
| **AI/ML 模型** | 支持在线训练和推理 |

### NWDAF 分析数据类型

| 数据类型 | 示例 | 分析用途 |
|---------|------|---------|
| **网络状态** | [[PRB]] 利用率、时延、抖动 | 切片负载均衡 |
| **用户级数据** | 吞吐量、附着成功率 | 体验优化 |
| **会话数据** | PDU Session 建立成功率 | [[SMF]] 策略优化 |
| **无线数据** | [[MR]]、[[RSRP]]、[[SINR]] | 覆盖优化 |
| **KPI** |掉线率、切换成功率 | [[KPI]] 分析 |

### 与其他网元的协同

| 协同网元 | 交互内容 |
|---------|---------|
| [[AMF]] | 订阅移动性事件、网络切片负载分析结果 |
| [[SMF]] | 订阅会话管理相关分析，QoS 策略优化 |
| [[PCF]] | 提供分析结果，触发策略更新 ([[QoS]] 调整) |
| [[NMS]] | 上报网络健康状态和告警 |
| [[UPF]] | 获取用户面流量统计数据 |

### 典型应用场景

| 场景 | 说明 |
|------|------|
| **切片资源动态调整** | NWDAF 分析发现某切片拥塞，通知 [[PCF]] 调整资源分配 |
| **用户体验优化** | 识别体验差的用户，触发 [[SMF]] 调整 [[QoS]] |
| **异常检测** | 检测 [[gNB]] 异常，通知运维 |
| **5G 语音优化** | 分析 [[VoNR]] 通话质量，优化切换参数 |

### 相关概念

- [[5GC]] - NWDAF 所在的 5G 核心网
- [[AMF]] / [[SMF]] / [[PCF]] - NWDAF 的服务消费者
- [[QoS]] - NWDAF 分析结果可触发 QoS 策略调整
- [[KPI]] - NWDAF 分析的基础数据之一
